Multi-stage Docker image

🏗️ Stage 1 · builder node:24-bookworm

  • Installs only the web + server workspace subgraph
  • Compiles native modules (node-pty, fff-node) against glibc
  • Builds web client + server bundle (vp run --filter t3 build)
  • Produces a pruned production tree via pnpm deploy --prod
→

📦 Stage 2 · runtime node:24-bookworm-slim

  • Slim glibc base + git, curl, CA certs
  • Installs provider CLIs: codex, claude-code, opencode, cursor-agent
  • Copies only dist/ + pruned node_modules/
  • Headless serve on :3773, state on /data volume
Key design characteristics

🔒 Stateful, single replica

  • SQLite + WebSocket + spawned child processes
  • min = max = 1 — no horizontal scaling
  • Rollback journal mode for Azure Files SMB compatibility

💾 Persistence model

  • /data (Azure Files): state, secrets, provider logins
  • /workspace: ephemeral, repos cloned per session

🛡️ Identity & secrets

  • User-assigned managed identity — no passwords in config
  • AcrPull for image, Key Vault Secrets User for API keys
  • OIDC federation for CI (no long-lived credentials)

🌍 Access control

  • Public HTTPS ingress gated by a bearer pairing token
  • Optional IP allowlist (allowed_ip_cidrs)
  • TLS 1.2+, insecure connections rejected
